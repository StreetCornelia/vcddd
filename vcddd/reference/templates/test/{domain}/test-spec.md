# {Domain} 黑盒功能测试规格

> 本文件由 TDD 桥梁 `reference/engine/tdd-bridge.md` 从 `docs/vcddd/design/{domain}/business.md` 机械推导。
> 每条测试标注来源，可追溯到 business.md 的具体行。
> 
> **类型**：黑盒测试 — 仅针对域对外能力（boundary.md 中定义的命令入口、事件消费接口、读模型）。
> **描述方式**：自然语言 / 伪代码，不包含具体编程语言代码。
> **覆盖要求**：全覆盖，不抽样。每条命令的所有路径、每个参数的边界情况都有测试。
> 
> **核心字段是「目标」**：目标是测试要验证的业务场景的详细描述，必须写得充分具体。
> 其余字段（前置/操作/期望/错误来源）是目标的环境支撑，如有遗漏实现 Agent
> 可基于目标中的业务描述自行推演补充。但目标本身必须完整，新 Agent 读完就能独立理解意图。

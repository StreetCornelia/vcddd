# 前端交互模型

## 核心原则

后端负责业务数据的合法性与状态流转；前端负责用户意图到业务数据的低成本生成过程。

前端可以辅助用户产生数据，但不能替代 `server/{domain}/` 做最终业务判断。

## 数据产生策略

| 数据类型 | 产生方式 | 降低输入成本的方法 | 最终校验归属 |
|----------|----------|--------------------|--------------|
| | 用户输入 / 默认值 / 上下文 / 历史记录 / 推荐 / 模板 / 扫描 / 批量选择 | | |

## 交互状态

| 状态 | 业务含义 | UI 表现 | 恢复方式 |
|------|----------|---------|----------|
| draft | | | |
| loading | | | |
| error | | | |
| confirming | | | |
| success | | | |

## 输入降本检查

每个页面设计前必须回答：

- 这个页面让用户产生什么业务数据？
- 哪些字段不应该让用户逐项手填？
- 哪些字段可以从上下文推断？
- 哪些字段可以用默认值、历史记录、推荐、模板、扫描或批量选择生成？
- 用户必须在哪些点明确确认？
- 失败后如何恢复、撤销或重试？

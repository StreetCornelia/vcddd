# TDD Bridge Reviewer 指令

审查生成的 test-spec.md 是否满足以下要求。**一次性给出全部问题，不允许分批抛出。**

## 核对清单

### 格式正确性
- [ ] 所有测试使用自然语言或伪代码描述，没有具体编程语言代码
- [ ] 每条测试包含：目标、前置条件、操作、期望结果

### 功能完整性
- [ ] 每条命令 → 至少 1 个正常路径测试
- [ ] 每个事件消费 → 至少 1 个测试
- [ ] 每个读模型 → 至少 1 个测试

### 分支完整性
- [ ] business.md 列出的每条失败路径 → 有对应测试（逐条核对，不漏）
- [ ] 幂等策略 → 有对应测试

### 数据边界性
- [ ] 每个字符串参数 → 空字符串/超长/非法格式
- [ ] 每个数值参数 → 0/负数/超上限
- [ ] 每个枚举参数 → 非法值
- [ ] 每个关联 ID → 不存在 ID/格式错误

### 一致性
- [ ] 所有错误测试写明了具体错误原因和错误来源
- [ ] business.md 未定义的没有自行补充（标记 [待补充]）

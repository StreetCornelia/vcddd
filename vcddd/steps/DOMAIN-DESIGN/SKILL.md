# Step: D¹ — Domain Design（域设计）

仅以确认的业务事实为输入，推导域边界、决策归属、不变式、事件与协作契约。

## 前置条件

- `docs/vcddd/requirements/facts.md` 已存在且经用户确认

## 输入

- `facts.md`（唯一的输入来源）

## 输出

每个域：`docs/vcddd/design/{domain}/boundary.md` + `business.md`

## 执行流程

1. **建立全局视图**：读 facts.md，建立业务事实全景
2. **识别域边界**：对每组事实追问——独立语言？独立不变式？独立演进？
3. **设计决策主权**：每个域记录哪些判断只能由它做出
4. **设计不变式**：每条必须有违反后果
5. **设计状态机**：合法迁移 + 非法迁移
6. **设计协作契约**：命令、事件、失败语义
7. 用户确认后写入文档

## 禁止

- 讨论数据库表结构
- 从技术模块反推域边界
- 把"调用关系"当作"协作契约"

## 如果卡住

读 `../../reference/guides/design.md` 获取完整流程。

## 日志

创建 `docs/vcddd/design/{domain}/progress.log` Task 1。每个域边界识别、每写入一个文档 → 追加记录。

## 验证

- [ ] 每个域能力边界明确
- [ ] 每条不变式有违反后果
- [ ] 状态机含合法和非法迁移
- [ ] 每个命令全部失败路径已列出
- [ ] 用户确认所有域设计后进入下一步

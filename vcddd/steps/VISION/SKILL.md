# Step: V — Vision（意图视野）

把用户的自然语言描述转化为可操作的原始输入文档。

## 前置条件

无（这是标准 VCDDD 流程的入口步骤）。

## 输入

- 用户的自然语言描述（模糊想法、口头表达、产品需求）

## 输出

`docs/vcddd/requirements/input.md`

## 执行流程

1. 忠实记录用户意图，不做删减或解释
2. 识别核心业务场景（3-7 个）
3. 明确产品边界（做什么、不做什么）
4. 记录用户的设计原则或价值取向（如有表达）

## 禁止

- 引入技术词汇
- 进行域分析或架构判断
- 推导到具体技术方案
- 把意图结构化为功能列表

## 如果卡住

读 `../../reference/guides/requirements.md` 获取完整流程。

## 日志

在 `docs/vcddd/progress.log` 追加 Task 节点，记录目标详细描述和执行操作。

## 验证

- [ ] 用户确认："这是我想做的东西"
- [ ] 没有任何技术词汇混入描述
- [ ] 没有进行任何域分析或架构判断

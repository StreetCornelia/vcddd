# Code Quality Reviewer 指令

审查 {domain} 域的代码是否符合 tech-stack.md 的全部约定。

## 第一步：运行测试

审查前先运行该域的全部测试。全部通过后才能进入核对清单。有失败则记录错误信息交给 Code Writer。

## 核对清单

## 核对清单

### 命名约定
- [ ] 类/函数/变量/文件命名符合约定
- [ ] 事件名用过去时，命令用动宾结构
- [ ] 状态名与 business.md 完全一致

### 目录与结构
- [ ] server/{domain}/ 包含该域全部内容
- [ ] 无 tech-stack.md 未规定的目录
- [ ] 每个文件单一职责

### 依赖
- [ ] 所有依赖在 tech-stack.md 中有记录
- [ ] 无未授权新依赖

### 日志
- [ ] 使用规定的日志库、JSON 格式
- [ ] 关键位置有日志：命令入口/状态迁移/失败/事件
- [ ] 日志包含必填字段

### 错误处理
- [ ] 错误分类符合约定
- [ ] 失败路径显式处理

### 事务
- [ ] 一个事务只操作一个聚合
- [ ] 事务内无外部调用

### 代码结构
- [ ] server/{domain}/ 无框架类型
- [ ] 状态迁移通过聚合方法完成

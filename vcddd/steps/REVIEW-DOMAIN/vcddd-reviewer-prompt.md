# VCDDD Compliance Reviewer 指令

审查 {domain} 域的 VCDDD 合规性。Stage 1 和 Stage 2 已通过，本层检查前两层未覆盖的 VCDDD 专项要求。

## 第一步：确认前序测试已运行

确认该域的全部测试（白盒+黑盒）已在 Stage 1（Spec Reviewer）验证通过。

**如果发现前序 Reviewer 未运行测试 → 拒绝执行，返回给控制器：**

```
拒绝执行：Stage 1 Spec Reviewer 未运行全部测试。
控制器必须先派遣 Spec Reviewer 补跑测试，才能继续 Stage 3。
```

**如果确认已运行全部测试 → 进入第二步。**

## 第二步：核对清单

### 定义准确性
- [ ] 域边界正确——代码未越界
- [ ] 通用语言一致——命名与 business.md 词表一致
- [ ] 决策主权未侵犯——其他域未直接修改本域状态
- [ ] Spec Reviewer 遗留问题合理

### 技术栈规则
- [ ] 域层与框架层分离——server/{domain}/ 无框架 import
- [ ] 框架层无业务逻辑
- [ ] 模块完整——含数据访问实现，无全局 infra/
- [ ] 日志规范遵守

### VCDDD 专项要求
- [ ] 事务与聚合边界对齐
- [ ] 状态迁移通过聚合方法（无直接字段赋值）
- [ ] 事件在事务提交后发布
- [ ] 跨域协作通过事件（无直接访问数据库）
- [ ] 幂等策略已实现

## 判定

✅ VCDDD 合规 — 通过
❌ 不合规 — 列出问题，返回控制器派遣 Implementer 修复
⚠️ 有条件合规 — 标记 DONE_WITH_CONCERNS
🚫 拒绝执行 — 前序 Reviewer 未运行测试，必须先补跑

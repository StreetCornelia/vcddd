# Spec Compliance Reviewer 指令

审查 {domain} 域的代码实现是否与 business.md 完全一致。

## 第一步：运行测试

审查前先运行该域的全部测试：

```bash
{tech-stack.md 中定义的测试命令}
```

- **全部通过** → 进入第二步
- **有失败** → 返回给控制器：列出每个失败测试的名称和错误信息，由控制器派遣 Implementer 修复。修复完成后控制器重新派遣你审查。

## 第二步：核对清单

### 聚合与不变式
- [ ] 每个聚合构造与 business.md 一致
- [ ] 每条不变式有强制代码（不依赖调用方自觉）
- [ ] 违反时返回的错误描述与 business.md 一致
- [ ] 无多余的不变式

### 状态机
- [ ] 每条合法迁移有对应代码
- [ ] 每条非法迁移被显式拒绝
- [ ] 迁移时产出对应的领域事件

### 命令处理
- [ ] 每条命令正常路径已实现
- [ ] 每条失败路径已实现（逐条核对）
- [ ] 失败路径错误分类与 business.md 一致
- [ ] 幂等策略已实现

### 事件
- [ ] 发布的事件与 boundary.md 契约一致
- [ ] 消费的事件有处理器
- [ ] 事件在事务提交后发布

# Step: C — Context（事实上下文）

把意图澄清为经用户逐条确认的业务事实。

## 前置条件

- `docs/vcddd/requirements/input.md` 已存在且经用户确认

## 输入

- 已确认的 `input.md`

## 输出

`docs/vcddd/requirements/facts.md`（经用户逐条确认）

## 执行流程

**按以下顺序逐步执行，每步完成后再进入下一步。**

### 第一步：五维翻译（按顺序执行）

用五个维度作为翻译滤镜，逐层把用户的口语化描述转化为结构化的业务事实：

1. **事实层**：翻译功能描述为事实描述（"用户可以下单"→"一笔待履约的交易承诺存在"）
2. **主权层**：找出每个业务判断的最终解释权归属（谁对这件事有最终解释权？）
3. **约束层**：识别必须同步保证的不变式（哪些约束必须在任何时候成立？）
4. **过程层**：识别需要状态化的长流程（哪些过程跨越时间？中途停止能说清停在哪里吗？）
5. **协作层**：识别跨域协作的事件（哪些变化会触发其他域的响应？）

### 第二步：逐条确认

把翻译后的事实逐行展示给用户：

1. 每展示一条事实，等待用户确认
2. 用户说"对"或等价肯定 → 写入 facts.md
3. 用户描述修改 → 按用户描述修改该条事实，重新展示，再次确认
4. **不暂停、不中断**：用户拒绝或修改时，根据用户反馈调整后继续下一条

### 第三步：完整性检查

全部事实确认后，逐条检查：
- 没有"我们假设"的表述残留
- 对每条事实问过："技术栈整体替换，这条规则还成立吗？"（成立的才是业务真相）

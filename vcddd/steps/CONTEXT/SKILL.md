# Step: C — Context（事实上下文）

把意图澄清为经用户逐条确认的业务事实。

## 前置条件

- `docs/vcddd/requirements/input.md` 已存在且经用户确认

## 输入

- 已确认的 `input.md`

## 输出

`docs/vcddd/requirements/facts.md`（经用户逐条确认）

## 执行流程

通过五维度翻译将意图转化为业务事实：

1. **事实层**：翻译功能描述为事实描述（"用户可以下单"→"一笔待履约的交易承诺存在"）
2. **主权层**：找出每个业务判断的最终解释权归属
3. **约束层**：识别必须同步保证的不变式
4. **过程层**：识别需要状态化的长流程
5. **协作层**：识别跨域协作的事件

逐条确认：每一行 facts.md 必须由用户逐条确认后写入。

## 如果卡住

读 `../../reference/guides/requirements.md` 获取完整流程。

- [ ] 对每条事实问过："技术栈整体替换，这条规则还成立吗？"

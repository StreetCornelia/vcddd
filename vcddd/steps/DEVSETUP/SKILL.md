# Step: D² — Dev Setup（技术栈确立）

确定技术栈并写入文档约定。与用户交互的优先级：

1. 用户有明确要求 → 直接遵循
2. 有现有代码 → 分析后向用户呈现，确认后写入
3. 全新项目 → 提供 2-3 个方案，用户选择后写入
4. 用户声明"不关心" → AI 自行选择后写入（标注原因）

## 前置条件

- 所有域的 `boundary.md` 和 `business.md` 已确认

## 输入

- 已确认的域设计文档 + 项目目录（如有代码）+ 用户偏好（如有）

## 输出

`docs/vcddd/tech-stack.md`

## 执行流程

1. 判断用户是否有明确要求 → 有则直接使用
2. 判断是否有现有代码 → 扫描配置文件 + 抽样读代码
3. 向用户呈现（标注 [配置文件] / [代码推断] / [社区约定] / [待确认]）
4. 用户确认后写入

## 如果卡住

读 `../../reference/engine/tech-setup.md` 获取完整流程。

## 日志

在 `docs/vcddd/progress.log` 追加 Task 节点。记录扫描到的技术栈信息和用户确认结果。

## 验证

- [ ] 用户已确认全部内容
- [ ] 所有约定已写入，无口头约定残留
- [ ] 有现有代码时与代码实际配置一致

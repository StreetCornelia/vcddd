# Step: D² — Dev Setup（技术栈确立）

确定技术栈并写入文档约定。

## 前置条件

- 所有域的 `boundary.md` 和 `business.md` 已确认

## 输入

- 已确认的域设计文档 + 项目目录（如有代码）+ 用户偏好（如有）

## 输出

`docs/vcddd/tech-stack.md`

## 执行流程

### 第一步：判断入口

```
用户是否有明确的技术栈要求？
    │
    ├── 有 → 直接遵循，不调研不推荐
    │     要求不完整时（如只指定语言，未指定框架/数据库）：
    │     → 针对缺失部分提供 2-3 个补充建议，由用户决定
    │
    └── 无 → 检查是否有现有代码？
          │
          ├── 有代码 → 读 `../../reference/engine/tech-setup.md` 的"有现有代码路径"
          │     → 按该文件的步骤扫描配置文件 + 抽样读代码
          │     → 向用户呈现分析结果
          │
          └── 无代码 → 提供 2-3 个可行技术栈方案（附适合理由和取舍）
                → 向用户呈现方案
```

### 第二步：呈现 + 确认循环

1. 向用户呈现技术栈草案（格式参考 `../../reference/engine/tech-setup.md`）
2. 等待用户确认
3. 用户确认 → 写入 tech-stack.md
4. 用户否决或有修改意见 → 基于反馈调整 → 重新呈现 → 循环直到用户确认
5. 用户声明"不管，你来定" → AI 自行选择后写入（标注"由 AI 自主选择"的原因）

## 如果卡住

读 `../../reference/engine/tech-setup.md` 获取完整流程。

## 日志

在 `docs/vcddd/progress.log` 追加 Task 节点。记录扫描到的技术栈信息和用户确认结果。

- [ ] 所有约定已写入，无口头约定残留
- [ ] 有现有代码时与代码实际配置一致
